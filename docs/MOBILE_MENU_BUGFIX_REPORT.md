# Mobile Menu Bugfix Report

Date: `2026-06-13`  
Preview affected: `https://noordtune-www.vercel.app/pl`

## Bug

On iPhone/mobile browser, the opened mobile menu visually overlapped with the homepage hero. Navigation links, language controls, hero headline, hero CTAs and badges appeared mixed together, making the page look broken and unsafe for launch.

## Root Cause

The previous mobile menu was rendered inside the sticky header and used a semi-transparent overlay. On mobile browsers this could remain in the header stacking context and visually blend with the hero underneath. The page also did not lock body scroll while the menu was open, and the floating WhatsApp button shared a similar z-index range with page chrome.

## Fix

- Rebuilt the mobile menu as a portal-rendered full-screen dialog mounted on `document.body`.
- Added `position: fixed`, `inset: 0`, `z-index: 120` and a solid premium black/red background.
- Added iOS safe-area padding.
- Added body scroll lock while the menu is open and restoration after close.
- Added Escape key close behavior.
- Added close-on-route-change behavior.
- Added close-on-navigation-link-click behavior.
- Kept all menu links localized for Dutch, English and Polish.
- Kept Power Catalog CTAs pointed to `https://power.noordtune.nl/`.
- Kept WhatsApp CTAs pointed to `https://wa.me/31685759600`.

- Page scroll is locked while the menu is open.
- The floating WhatsApp button sits below the menu overlay and does not overlap it.
- Polish labels retain correct diacritics.
- Close button, Escape key and link clicks close the menu.
